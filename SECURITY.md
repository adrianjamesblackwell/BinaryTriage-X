# Security Policy

## Reporting a Vulnerability

Please do not disclose potential vulnerabilities through public GitHub issues.

Security reports should include:

- affected component
- observed behavior
- reproduction conditions
- potential impact
- proposed mitigation, when available

Do not include live malware, credentials, private data, or weaponized proof-of-concept material in public reports.

## Supported Versions

BinaryTriage-X is currently under active pre-release development.

Security support policies will be formalized before the first stable public release.
