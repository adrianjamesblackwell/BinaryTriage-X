# Architecture Decision Records

## Purpose

This directory contains the Architecture Decision Records for BinaryTriage-X.

An Architecture Decision Record documents an important engineering decision made during the design and development of the platform.

Each ADR explains:

- what problem was being solved
- what options were considered
- which option was selected
- why the decision was made
- what trade-offs were accepted

The goal is to preserve engineering reasoning over time.

---

## Why ADRs Exist

Code shows what the system does.

ADRs explain why the system was designed that way.

Without ADRs, future maintainers may understand the implementation but not the reasoning behind it.

BinaryTriage-X uses ADRs to prevent architectural knowledge from being lost.

---

## ADR Status Values

Each ADR must use one of the following status values:

- Proposed
- Accepted
- Deprecated
- Superseded

Only Accepted ADRs represent current architecture.
